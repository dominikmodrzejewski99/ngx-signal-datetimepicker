A date & time picker that thinks in signal(), not subscribe.

ngx-signal-datetimepicker is a standalone Angular component built on top of the new Signal Forms API. No RxJS wiring, no FormControl ceremony — just a reactive primitive that drops into any template and stays in sync with your state.

Six things every Angular team kept rebuilding. Now one import.

01 Primitive

Two-way model()

Bind a writable signal in either direction. No ngModel required, no value accessors to write.

02 Reactivity

Computed-friendly

Read value() inside any computed or effect. Change detection just works.

05 / Compatibility & FAQ

Will it work in my stack?

Angular versions
  • Angular 21.xv0.1+✓ tested
  • Angular 22.x (when released)—✓ planned
  • Angular ≤ 20.x—unsupported (no Signal Forms)
Environment
  • Signal Forms ([formField])—✓ first-class
  • Reactive Forms ([formControl])—✓ ControlValueAccessor
  • Zoneless change detection—✓ signals all the way
  • Standalone components—✓ required
Why not just use Angular Material's datepicker?
Material's picker is excellent but built around FormControl and value accessors. If your codebase has moved to signals and zoneless change detection, the impedance mismatch shows up fast: every interaction round-trips through valueChanges. ngx-signal-datetimepicker is just a model() — what you read is what's selected, synchronously. It also ships date + time in one control, which Material's pair (Datepicker + a separate time input) doesn't.
Does it work with Reactive Forms?
Yes — the picker implements ControlValueAccessor, so it drops into [formControl], formControlName, or [(ngModel)] like a native input. disable() / enable() on the FormControl are honoured via setDisabledState(); closing the panel marks the control as touched.
What's the date library? date-fns? Day.js? Luxon?
None of them. The picker uses the native Date object plus a small internal calendar engine. If you want to format with date-fns or Luxon in your app, nothing stops you — but you don't have to install anything. Bundle stays at ~44 kB gzipped.
Can I theme it?
Override CSS custom properties — --ngx-dt-accent, --ngx-dt-focus, --ngx-dt-target-size, --ngx-dt-radius, and ~15 more. No SCSS compilation, no ::ng-deep. The component scopes its styles with :host so you can target it from any parent.
What about timezones?
The [timeZone] input accepts any IANA name (Europe/Warsaw, America/New_York, UTC). The calendar and time inputs render wall-clock time in that zone while the bound Date stays a UTC instant — so serialisation is unambiguous and DST transitions are handled by Intl.
How accessible is it, really?
The defaults target WCAG 2.2 Level AAA. Every text token is ≥7:1 contrast on the default surface, every interactive target is ≥44×44 CSS px, the focus indicator is 3 px solid with ≥3:1 contrast, the calendar grid uses role="grid" with aria-selected, and arrow / Home / End / PageUp / PageDown / Enter / Space all work. Focus moves into the panel on open and back to the trigger on close.
